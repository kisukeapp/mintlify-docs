The CLI is the headless version of Kisuke Connect — the same daemon, without the desktop GUI. Use it for servers, containers, and automated environments.

## CLI

### Install via npm

```bash
npm install -g @kisuke/cli
```

### Install via binary download

Download the binary directly for your platform:

<Tabs>
  <Tab title="Linux x64">
    ```bash
    curl -L https://kisuke.app/releases/latest/kisuke-connect-linux-amd64 -o kisuke-connect
    chmod +x kisuke-connect
    sudo mv kisuke-connect /usr/local/bin/
    ```
  </Tab>
  <Tab title="Linux ARM64">
    ```bash
    curl -L https://kisuke.app/releases/latest/kisuke-connect-linux-arm64 -o kisuke-connect
    chmod +x kisuke-connect
    sudo mv kisuke-connect /usr/local/bin/
    ```
  </Tab>
  <Tab title="macOS">
    ```bash
    curl -L https://kisuke.app/releases/latest/kisuke-connect-darwin-universal -o kisuke-connect
    chmod +x kisuke-connect
    sudo mv kisuke-connect /usr/local/bin/
    ```
  </Tab>
  <Tab title="Windows">
    ```powershell
    Invoke-WebRequest -Uri https://kisuke.app/releases/latest/kisuke-connect-windows-amd64.exe -OutFile kisuke-connect.exe
    ```
  </Tab>
</Tabs>

### Authenticate

```bash
kisuke-connect login
```

### Usage

```bash
kisuke-connect up        # Start the daemon
kisuke-connect status    # Check status
kisuke-connect devices   # List connected devices
kisuke-connect down      # Stop the daemon
```

## Docker

### Quick start

```bash
docker run -d \
  --name kisuke-connect \
  --restart unless-stopped \
  -v kisuke-data:/var/lib/kisuke \
  -e KISUKE_AUTH_KEY=your-auth-key \
  kisuke/kisuke-connect
```

### Docker Compose

```yaml
version: '3.8'

services:
  kisuke-connect:
    image: kisuke/kisuke-connect:latest
    container_name: kisuke-connect
    restart: unless-stopped
    volumes:
      - kisuke-data:/var/lib/kisuke
    environment:
      - KISUKE_AUTH_KEY=${KISUKE_AUTH_KEY}
    network_mode: host

volumes:
  kisuke-data:
```

### Auth keys

Generate an auth key from your Kisuke mobile app:

1. Open Kisuke on your mobile device
2. Go to **Settings** > **Auth Keys**
3. Tap **Generate New Key**
4. Copy the key and use it in your Docker configuration

<Warning>
Auth keys provide access to your Kisuke network. Keep them secure and rotate them regularly.
</Warning>

## Environment variables

| Variable | Description | Default |
|----------|-------------|---------|
| `KISUKE_AUTH_KEY` | Authentication key for headless login | — |
| `KISUKE_HOSTNAME` | Custom hostname for this device | Container ID |
| `KISUKE_STATE_DIR` | Directory for state files | `/var/lib/kisuke` |
