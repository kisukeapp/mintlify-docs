---
title: "Install CLI"
description: "Install Kisuke Connect via CLI for servers and headless environments"
---

The CLI is the headless version of Kisuke Connect — the same daemon, without the desktop GUI. Use it for servers and automated environments.

## Install via npm

```bash
npm install -g @kisuke/cli
```

## Install via binary

Download the binary directly for your platform:

<Tabs>
  <Tab title="Linux x64">
    ```bash
    curl -L https://kisuke.app/releases/latest/kisuke-connect-linux-amd64 -o kisuke-connect
    chmod +x kisuke-connect
    sudo mv kisuke-connect /usr/local/bin/
    ```
  </Tab>
  <Tab title="Linux ARM64">
    ```bash
    curl -L https://kisuke.app/releases/latest/kisuke-connect-linux-arm64 -o kisuke-connect
    chmod +x kisuke-connect
    sudo mv kisuke-connect /usr/local/bin/
    ```
  </Tab>
  <Tab title="macOS">
    ```bash
    curl -L https://kisuke.app/releases/latest/kisuke-connect-darwin-universal -o kisuke-connect
    chmod +x kisuke-connect
    sudo mv kisuke-connect /usr/local/bin/
    ```
  </Tab>
  <Tab title="Windows">
    ```powershell
    Invoke-WebRequest -Uri https://kisuke.app/releases/latest/kisuke-connect-windows-amd64.exe -OutFile kisuke-connect.exe
    ```
  </Tab>
</Tabs>

## Authenticate

```bash
kisuke-connect login
```

## Usage

```bash
kisuke-connect up        # Start the daemon
kisuke-connect status    # Check status
kisuke-connect devices   # List connected devices
kisuke-connect down      # Stop the daemon
```

## Environment variables

<ParamField body="KISUKE_HOSTNAME" type="string">
  Custom hostname for this device.
</ParamField>

<ParamField body="KISUKE_STATE_DIR" type="string" default="/var/lib/kisuke">
  Directory for state files.
</ParamField>
